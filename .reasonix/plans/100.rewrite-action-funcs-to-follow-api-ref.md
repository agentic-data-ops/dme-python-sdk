# 计划：重写 Action 函数的描述、参数和响应以匹配 API 参考文档

## 目标

遍历 `pydme/actions/*.py` 中每个函数，通过函数中使用的 **HTTP方法 + URI** 在 `reference/dme-api-reference.md` 中找到对应的 API 定义，然后更新该函数的参数描述、参数校验和响应描述，使其与参考文档一致。

---

## 执行方式

**逐个文件、逐个函数处理。** 每处理一个函数就立即更新其 docstring 和参数校验逻辑。

---

## 处理步骤

### 对每个函数，执行以下操作：

#### 第1步：提取方法的 URI

从函数体中提取：
- `url = "/rest/xxx/yyy"` 或类似语句 → URI
- HTTP 方法（GET/POST/PUT/DELETE）→ 在函数签名的注释中标记，或从函数名称推断（`create_*`=POST, `delete_*`=DELETE, `list_*`=GET, `modify_*`=PUT）

**查找规则：**
1. 从函数体中搜索 `url =` 或 `self.url =` 赋值语句
2. 如果没有显式 URL，检查函数是否封装了 `client.get/post/put/delete()` 调用
3. 如果函数名暗示了某个操作（如 `login` 对应 `/rest/plat/smapp/v1/sessions`），通过名称推断

#### 第2步：在参考文档中查找匹配的 API 定义

用提取的 **HTTP方法 + URI** 在 `reference/dme-api-reference.md` 中搜索。

**搜索策略：**
1. 在参考文件中搜索 `- **URI**: <匹配的URI>` 所在行
2. 从该行向上找到 `## <名称>` 确定 API 定义块的起始位置
3. 从该行向下读取完整的 API 定义块（直到下一个 `##` 为止）
4. 解析该 API 块中的：
   - `**描述**` → 函数描述
   - `**路径参数**` 表格 → 路径参数
   - `**查询参数**` 表格 → 查询参数
   - `**请求 Body 参数**` 表格 → Body 参数
   - `**响应字段**` 表格 + `**响应引用对象**` → 响应结构
   - `**状态码**` → 状态码说明

**匹配优先级：**
- 精确 URI 匹配优先
- 路径参数模板匹配（如 `/rest/storages/{storage_id}` 匹配 `/rest/storages/abc123`）
- 找不到时标记为 `⚠️ 未找到匹配`，保留现有实现

#### 第3步：重写函数 docstring

按照参考文档的 API 定义重写 docstring 的三个部分：

**描述部分：**
```
"""
<参考文档中的**描述**内容>

Args:
    client: DME API 客户端
    <param1>: <说明> (<是否必选>, <类型>, <值域约束>)
    
Returns:
    <响应结构说明>
"""
```

**参数描述格式：**
```
参数名: 说明文字 (是否必选, 类型, 值域约束)。可选值：枚举值(描述), ...
```

参考 `pydme` 项目约定的 `parse_docstring` 格式（参见项目记忆中的函数参数帮助文档格式约定）。

**参数名映射规则：**
- 参考文档使用 `user-name`（带连字符）→ 函数参数用 `user_name`（下划线）
- 参考文档使用 `page_size` → 保持 `page_size`
- List 类型的参数保持 `List<type>` 标注

#### 第4步：更新参数校验逻辑

在函数体中，根据参考文档的约束添加参数校验：

**必选参数检查：**
```python
if not param_name:
    raise ValueError("参数名 是必选参数")
```

**值域检查：**
```python
if param_value not in ['enum1', 'enum2']:
    raise ValueError(f"参数名 可选值：enum1, enum2")
```

**长度/范围检查：**
```python
if len(param_value) > max_len:
    raise ValueError(f"参数名 最大长度：{max_len}个字符")
```

#### 第5步：更新函数返回值的 docstring

从参考文档的 **响应字段** 和 **响应引用对象** 解析返回结构：

```
Returns:
    {
        total: 总数 (int32),
        datas: 数据列表 (List<UserData>)。参数格式如下：[{
            id: 用户ID (integer, 1~2147483647),
            name: 用户名 (string, 6~32个字符),
        }, ...]
    }
```

遵循项目约定的 Returns 格式：**直接以 `{` 开头**，不加前缀文字。

## 验证方式

每个文件处理完成后：
1. `python -c "from pydme.client import DMEAPIClient; from pydme.actions.<file> import *"` — 无导入错误
2. `pydme <topic> --help` — CLI 参数描述正确显示
3. 检查关键参数的 --help 输出，看描述是否与参考文档一致

---

## 注意事项

1. **URI 精确匹配**：不同版本（v1/v2）的同一接口视为不同API
2. **保留客户端调用逻辑**：只改 docstring 和参数校验，不改 API 调用逻辑
3. **路径参数转换**：URL 中的 `{storage_id}` 对应函数参数 `storage_id`
4. **无匹配处理**：函数 URI 在参考文档中找不到时，标记并保持现有实现
5. **上下文管理**：每次处理 1-2 个文件后提交，避免上下文超限
